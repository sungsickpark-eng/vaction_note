
# 여행 일지 다이어리 — 기획 초안 통합본

> 세 가지 AI 기획안(Gemini · ChatGPT · Claude)을 종합하여 작성한 최종 기획 문서입니다.

---

## 1. 프로젝트 개요

**서비스명:** 여행 일지 다이어리

**한 줄 정의:** 구글 지도 위에서 여행 계획부터 실시간 메모, 사진 기록까지 한 번에 관리하는 타임라인 기반 여행 다이어리

**핵심 가치:**

|가치|설명|
|---|---|
|시각화|지도 중심의 직관적인 동선 확인|
|실시간성|이동 중에도 끊김 없는 가벼운 메모 및 기록|
|확장성|API 분리 구조로 웹 → 모바일 앱 전환이 용이한 설계|
|기록의 연속성|장소 · 시간 · 이동 경로 · 메모 · 사진을 하나로 연결|

---

## 2. 주요 사용자 (페르소나)

### 여행 계획자

여행 전에 방문할 장소를 정리하고 이동 순서와 동선을 미리 계획하고 싶은 사용자.

### 여행 기록자

여행 중 느낀 점, 맛집, 풍경, 순간의 감정을 현재 위치 기반으로 빠르게 기록하고 싶은 사용자.

### 콘텐츠 제작자

여행 후 블로그, 유튜브, 인스타그램 등에 올릴 여행 기록을 지도 기반으로 정리하고 싶은 사용자.

---

## 3. 핵심 기능 정의

### 3-1. 여행 일지 생성

사용자는 새로운 여행 일지를 생성하고 기본 정보를 입력한다.

- 여행 제목 / 기간 / 여행 지역
- 동행자 / 여행 설명 / 커버 사진
- 공개 여부 (나만 보기 / 링크 공유 / 전체 공개)

**예시:**

> 제주도 3박 4일 가족여행  
> 2026년 7월 10일 ~ 7월 13일  
> 제주공항 → 협재해수욕장 → 성산일출봉 → 서귀포

---

### 3-2. 구글 지도 연동 및 이동 계획

- 장소 검색 (Google Places Autocomplete)
- 지도에 핀 추가 및 방문 예정 시간 입력
- 장소 순서 변경 (드래그 앤 드롭)
- 날짜별 일정 분리 및 타임라인 UI
- 도보 · 자동차 · 대중교통 이동 방식 선택
- Google Maps Directions API로 경로 시각화 (Polyline 연결)
- 예상 이동 시간 / 거리 자동 계산
- 하루 일정 보기 / 전체 여행 경로 보기

**Day 1 예시:**

```
1. 제주공항
2. 렌터카 수령
3. 협재해수욕장  (이동: 차량 45분)
4. 숙소 체크인   (이동: 차량 20분)
5. 흑돼지 맛집   (이동: 도보 10분)
```

---

### 3-3. 이동 중 메모 기능

- 현재 위치 자동 태그 (GPS 좌표 → 장소명 역지오코딩)
- 텍스트 메모 + 감정 태그 (😊 😴 🍽 📸 등 커스텀 태그)
- 장소 태그 연결 / 작성 시간 자동 기록
- 지도 위에 메모 핀 오버레이로 표시
- 오프라인 임시 저장 → 온라인 복귀 시 자동 동기화

**메모 예시:**

> 바람이 정말 시원했다.  
> 해가 지는 시간이 너무 예뻐서 사진을 많이 찍었다.

---

### 3-4. 이동 중 사진 저장

- 카메라 직접 촬영 또는 갤러리 업로드
- EXIF GPS 데이터 자동 추출 → 지도에 자동 핀 생성
- 사진 촬영 시간 / 위치 자동 기록
- 사진별 캡션 및 태그 작성
- 장소별 사진 모아보기
- S3 업로드 + CDN 서빙 (썸네일 자동 생성, WebP 변환)
- 향후 앱 버전: 스마트폰 네이티브 카메라 직접 연동

---

### 3-5. 여행 일지 보기

여행이 끝난 후 전체 기록을 다양한 방식으로 확인한다.

|보기 방식|설명|
|---|---|
|지도 보기|전체 이동 경로와 핀을 지도에서 확인|
|날짜별 보기|일별 메모 + 사진 타임라인|
|장소별 보기|방문 장소 단위로 기록 정리|
|사진 앨범|그리드 뷰 + 지도 클러스터링|
|타임라인|시간 순서대로 메모 · 사진 스크롤|

## 4. 기술 스택

### 프론트엔드

|영역|기술|비고|
|---|---|---|
|프레임워크|Next.js 14 (App Router)|CSR 중심, 공유 페이지는 SSR/ISR로 SEO 대응|
|언어|TypeScript||
|스타일|Tailwind CSS||
|상태관리|React Query + Zustand|서버/클라이언트 상태 분리|
|지도|Google Maps JS SDK|`@vis.gl/react-google-maps`|
|오프라인|IndexedDB (idb 라이브러리)|임시 저장 및 sync|

### 백엔드

|영역|기술|비고|
|---|---|---|
|프레임워크|FastAPI (Python 3.11)|비동기, Pydantic v2, Swagger 자동 문서화|
|ORM|SQLAlchemy async + Alembic|DB 마이그레이션 관리|
|인증|JWT + Google OAuth2|`python-jose`|
|실시간|WebSocket (FastAPI)|위치 공유, 알림|
|공간 데이터|PostGIS (PostgreSQL 확장)|위치 기반 쿼리 최적화|

### 데이터베이스 & 스토리지

|영역|기술|비고|
|---|---|---|
|DB|SQLite (개발) → PostgreSQL + PostGIS (운영)|위치 데이터 처리 최적화|
|파일|Cloudflare R2 또는 AWS S3|presigned URL 직접 업로드|
|캐시|Redis|세션 · 지도 타일 캐싱|

### 배포

|영역|기술|
|---|---|
|Frontend|Vercel|
|Backend|Docker + AWS / GCP / Railway|
|Database|Supabase PostgreSQL 또는 AWS RDS|

## 7. 화면 구성

|#|화면|주요 기능|
|---|---|---|
|1|**메인 (랜딩)**|서비스 소개, 시작하기 버튼, 인기 공개 여행 일지|
|2|**대시보드**|내 여행 목록 (진행 중 / 완료), 새 여행 만들기|
|3|**여행 생성 / 편집**|제목, 기간, 지역, 커버 사진, 공개 여부 설정|
|4|**일정 계획**|좌측: 타임라인 패널 / 우측: 구글 지도 + 경로 미리보기|
|5|**여행 중 뷰**|현재 위치 트래킹, 빠른 메모 FAB, 카메라 버튼 (모바일 최적화)|
|6|**타임라인 뷰**|날짜별 메모 + 사진을 지도 핀과 함께 스크롤|
|7|**앨범 뷰**|사진 그리드 + 지도 클러스터링|
|8|**공개 일지 탐색**|공개된 여행 일지 검색 및 둘러보기|

> **레이아웃 방향:**
> 
> - 웹(플래너 화면): 좌측 스플릿 창(타임라인) + 우측 구글 지도
> - 모바일(이동 중 기록): 지도는 상단 소형 배치, 메모/카메라 버튼 대형 배치

## 8. API 엔드포인트 (주요)

```
# 인증
POST   /api/auth/google              Google OAuth 로그인
POST   /api/auth/refresh             토큰 갱신
DELETE /api/auth/logout              로그아웃

# 여행
GET    /api/trips                    내 여행 목록
POST   /api/trips                    여행 생성
GET    /api/trips/{id}               여행 상세
PUT    /api/trips/{id}               여행 수정
DELETE /api/trips/{id}               여행 삭제

# 일정 / 경유지
GET    /api/trips/{id}/days                      일별 일정 목록
POST   /api/trips/{id}/days/{day}/waypoints      경유지 추가
PUT    /api/waypoints/{id}                       경유지 수정 (순서 포함)
DELETE /api/waypoints/{id}                       경유지 삭제

# 메모
GET    /api/trips/{id}/memos         메모 목록 (지도 핀 포함)
POST   /api/trips/{id}/memos         메모 생성
PUT    /api/memos/{id}               메모 수정
DELETE /api/memos/{id}               메모 삭제

# 사진
POST   /api/photos/presign           S3 presigned URL 발급
POST   /api/photos                   사진 메타데이터 등록
GET    /api/trips/{id}/photos        여행 사진 목록
DELETE /api/photos/{id}              사진 삭제

# 지도 (Google Maps 서버 프록시)
GET    /api/maps/search?q=           장소 검색
GET    /api/maps/directions          경로 계산
GET    /api/maps/geocode?lat=&lng=   역지오코딩 (좌표 → 장소명)

# 공유
GET    /api/trips/{id}/share         공유 링크 발급
GET    /api/public/trips             공개 여행 일지 목록
```

## 10. 주요 기술적 고려사항

### Google Maps API 키 보안

- 프론트엔드 키: HTTP Referrer 제한 (도메인 화이트리스트)
- 백엔드 키: 서버사이드 전용, 환경변수 관리, IP 화이트리스트 적용
- 일일 쿼터 모니터링 및 알림 설정

### 렌더링 전략

- 지도/기록 화면: CSR (빠른 인터랙션)
- 공개 여행 일지 페이지: SSR / ISR (SEO 대응)

### 사진 업로드 최적화

- 클라이언트 → S3 직접 업로드 (presigned URL) — 백엔드 부하 최소화
- Lambda / Cloudflare Worker로 썸네일 자동 생성 (WebP 변환)
- EXIF 메타데이터 파싱은 클라이언트에서 처리 후 서버 전송

### 오프라인 지원

- 웹: IndexedDB에 임시 메모 저장 → 온라인 복귀 시 자동 sync
- 앱: SQLite 로컬 DB → 서버 양방향 동기화 (last-write-wins + 충돌 해결)

### 모바일 확장 준비 원칙

- API는 처음부터 REST + JSON 설계, 웹 전용 로직 백엔드에 포함 금지
- 비즈니스 로직은 custom hooks / service layer로 분리 (React Native 재사용)
- 환경변수로 플랫폼 분기 처리

### 공간 데이터 처리

- PostgreSQL + PostGIS 도입으로 반경 내 메모 검색, 경로 최적화 쿼리 지원
- 위치 데이터는 `GEOGRAPHY(POINT, 4326)` 타입 사용

### 성능 목표

|항목|목표|
|---|---|
|지도 초기 로딩|2초 이내|
|메모 저장 응답|500ms 이내|
|사진 썸네일 표시|1초 이내 (CDN 캐시 기준)|
